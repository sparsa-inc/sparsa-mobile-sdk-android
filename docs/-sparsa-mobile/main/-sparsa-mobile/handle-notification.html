handleNotification

fun handleNotification(payload: Map<String, String>, onDelete: () -> Unit)

Handles incoming notification payload from a push notification.

This method processes notification data received from push notifications (e.g., Firebase Cloud Messaging). It extracts relevant information from the payload and performs appropriate actions based on the notification type. For example, it can handle device removal notifications, credential verification requests, etc.

Parameters

payload

A map containing the data received with the notification. Keys and values are both strings.

onDelete

A callback function that is invoked when this device is removed from the digital address by another device. This allows the application to perform cleanup or UI updates when the device is unlinked.

Throws

SparsaError

if the notification processing fails or the payload contains invalid data.

fun handleNotification(extras: Bundle?, onDelete: () -> Unit)

Handles incoming notification payload from an Android Bundle.

This is an overloaded version of handleNotification that accepts an Android Bundle instead of a Map. It extracts string key-value pairs from the Bundle and processes them as notification data. This method is particularly useful when handling notifications received through Android's notification system, where data is typically provided as a Bundle.

Parameters

extras

The Bundle containing notification data, typically obtained from an Intent or notification extras.

onDelete

A callback function that is invoked when this device is removed from the digital address by another device. This allows the application to perform cleanup or UI updates when the device is unlinked.
